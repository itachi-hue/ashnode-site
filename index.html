The Problem

RAG gives you documents.
Not what is currently true.

Long-lived agents accumulate stale, conflicting, and superseded facts. Plain vector search has no concept of "current" — and that causes hard-to-reproduce failures in exactly the domains where it matters most.

Plain RAG
  • Returns all versions of a fact equally — old and new
  • Silently passes contradictions to the LLM
  • No concept of supersession or what "current" means
  • No audit trail — why did the agent say that?
  • Non-reproducible — same query, different context
  • No signal when retrieved context is incomplete
Ashnode
  • Superseded facts automatically excluded from recall
  • Contradictions surfaced within the retrieval neighborhood
  • Every item carries timestamp, decay score, and provenance
  • Full audit trail — every agent decision is inspectable
  • Deterministic — same query + policy = identical packet
  • Explicit completeness flag when a cap is hit
Core Features

Built for agents that run
in production for months

Every design decision in Ashnode addresses a real failure mode of long-lived agents in regulated environments.

🔁
Supersession
Mark facts as replaced when new ones arrive. Recall returns only what is currently true. Trend queries can optionally include the full supersession chain with timestamps.
⚡
Contradiction Detection
Background brain scans for conflicting facts within each recall neighborhood — heuristic and NLI-based — without blocking ingestion or retrieval.
📋
Full Provenance
Every item in a context packet carries source, timestamp, decay score, supersession status, and sequence number. Every agent decision is auditable and reproducible.
⏱️
Time Decay
Configurable half-life (tau) per item. Facts age out naturally. A drug interaction warning from last week outweighs one from two years ago — without manual curation.
🎯
Deterministic Recall
Same query + policy + store revision = identical context packet, every time. Zero non-determinism failures across all benchmark runs. Reproduce any agent decision exactly.
📦
Bounded Context Packets
Policy-controlled caps on items, contradictions, beliefs, and edges. Agents receive a principled slice — never everything, always the right things. Explicit flag when a cap is hit.
Quick Start

Three calls.
Structured memory.

Install, ingest facts, recall a bounded context packet. No infrastructure to manage — runs fully local, CPU-only.

Get Access to the Repo 
from ashnode import Ashnode

# Initialize with a local SQLite store
memory = Ashnode("agent_memory.db")

# Ingest facts as they arrive
memory.ingest(
    content="Patient BP is 102 mmHg",
    claim_key="patient_bp",
    tau=30.0  # 30-day half-life
)
memory.ingest(
    content="Patient BP is 110 mmHg",
    claim_key="patient_bp"  # auto-supersedes previous
)

# Recall returns a bounded context packet
packet = memory.recall("What is the patient's BP?")

# packet.items       → current facts only
# packet.contradictions → flagged conflicts
# packet.freshness   → aggregate decay score
# packet.items_truncated → completeness flag
